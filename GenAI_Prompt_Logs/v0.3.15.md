Cause found:

- Chromium root-level folders such as **Bookmarks bar** and **Other bookmarks** use `parentId: "0"`.
- The Details pane’s **Parent folder** dropdown only listed valid movable folder targets, so `"0"` was not present.
- That made the dropdown blank, which differed from the saved `detailsOriginal.parentId`, so it falsely marked **Parent folder** as modified.

Changes applied:

- Added a disabled **Browser root** placeholder option for root-level folders.
- Preserved the saved parent value instead of leaving the dropdown blank.
- Prevented the false **Unsaved changes** popup when switching to root-level folders.
