### What caused the warning

In v0.7.30, the embedded Options iframe was given:

```JavaScript
frame.sandbox = "allow-forms allow-same-origin allow-scripts";
```

Chromium warns about this because `allow-scripts` plus `allow-same-origin` can make the sandbox ineffective: a same-origin scripted page may be able to remove or bypass the sandbox restrictions.

In SBM’s case, the embedded Options page is already a packaged extension page:

```JavaScript
chrome.runtime.getURL("options.html?embedded=1")
```

So the sandbox was not giving meaningful extra protection, but it did create a browser security warning.

### Fix applied

Removed the iframe `sandbox` attribute entirely.

Kept:

```JavaScript
frame.referrerPolicy = "no-referrer";
frame.src = api.runtime.getURL("options.html?embedded=1");
```

Also kept the existing manifest CSP from v0.7.30, including:

```JSON
"frame-src 'self'"
```

### Why this is the safer fix

The Options iframe needs normal extension-page capabilities for:

- `chrome.storage`
- extension-local scripts/modules
- same-origin access to the manager warning/error log helpers

Removing `allow-same-origin` would likely break embedded Options behavior. Keeping both `allow-scripts` and `allow-same-origin` causes Chromium’s warning. So the clean fix is to avoid a misleading sandbox and rely on the existing constraints:

- packaged extension URL only
- `frame-src 'self'`
- `script-src 'self'`
- no remote assets or code
- `referrerPolicy="no-referrer"`
