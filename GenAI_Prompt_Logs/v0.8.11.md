### Summary of changes

#### 1. Fixed Library pane resizer layering

The Library pane resizer now renders above the sticky middle-pane column headers.

Changed the pane resizer to use:

```CSS
position: relative;
z-index: 4;
```

The middle column header uses a lower z-index, so the resize bar now visually sits on top instead of letting the header edge appear above it.

#### 2. Added middle-click bookmark opening in the middle pane

Middle-click now works on bookmark rows in the middle pane.

Behavior:

- Middle-clicking a single bookmark opens that bookmark.
- If there is an active middle-pane multi-selection and the clicked row is part of that selection, middle-click opens the selected direct bookmarks.
- Folders and separators are ignored for this gesture.
- URL opening still goes through SBM’s existing URL protection checks for `javascript:`, `data:`, and `blob:`.

#### 3. Added search result “Open Folder containing the Bookmark.”

When viewing search results, right-clicking a bookmark now shows:

```
Open Folder containing the Bookmark.
```

When clicked, SBM:

- exits search mode,
- navigates to the bookmark’s parent folder,
- selects the bookmark in that folder,
- scrolls the selected bookmark into view.

This action only appears for bookmark rows in search results.

#### 4. Security and performance notes

- No `innerHTML` or unsafe DOM injection was added.
- New UI text uses the existing i18n system.
- Bookmark opening is routed through the existing URL filtering/protection path.
- The search containing-folder action uses existing in-memory bookmark nodes and does not perform remote requests.
- The resizer fix is CSS-only and does not add runtime work.
- Middle-click handling is attached during existing row rendering, with no extra global scans.
