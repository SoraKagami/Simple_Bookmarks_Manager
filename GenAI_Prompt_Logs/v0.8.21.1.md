For SBM, I recommend standardizing on **LF** going forward.

Chromium extensions are web-style projects: `.js`, `.html`, `.css`, `.json`, `.md`, and locale files are all perfectly fine with LF on Windows, and LF avoids unnecessary churn when files are handled by Linux/macOS tooling, GitHub, CI, packaging scripts, or ChatGPT/container tooling.

Suggested repo rule:

```gitattributes
# Normalize extension/source text files to LF.
* text=auto eol=lf

# Keep binary files untouched.
*.png binary
*.jpg binary
*.jpeg binary
*.gif binary
*.webp binary
*.ico binary
*.zip binary
*.woff binary
*.woff2 binary
```

If you later add Windows-only helper scripts, those can be exceptions:

```gitattributes
*.bat text eol=crlf
*.cmd text eol=crlf
```
