### Changelog interface

- Added a new hamburger-menu item below **About** called **Changelog**.
- Clicking it opens a scrollable in-SBM modal.
- The modal fetches packaged `CHANGELOG.md`.
- Markdown is rendered into DOM nodes instead of using `innerHTML`.

Supported formatting includes:

- headings
- bullet lists
- paragraphs
- inline code
- fenced code blocks
- bold text
- safe HTTP/HTTPS markdown links

### Security considerations

- No `innerHTML` was added.
- About content is loaded only from a packaged extension page.
- Changelog markdown is rendered with `textContent` and DOM node creation.
- Markdown links are only activated for parseable `http:` or `https:` URLs.
- Dialog iframes keep `referrerPolicy = "no-referrer"`.
- Existing manifest CSP remains in place.

### Performance considerations

- About and Changelog content are created lazily only when the menu items are clicked.
- Changelog markdown text is fetched once per manager-page lifetime and cached for later openings.
- Modal content is cleared when closed to avoid keeping unnecessary iframe/DOM state alive.
