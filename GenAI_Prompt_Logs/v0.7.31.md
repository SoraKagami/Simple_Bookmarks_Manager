## What changed

### 1. Restored support for browser-internal bookmark URLs

The v0.7.30 hardening was too strict because it only allowed:

```JavaScript
http:
https:
ftp:
```

That blocked legitimate browser/bookmark use cases like:

```
about:config
chrome://settings/content/all
brave://settings/content/all
edge://...
opera://...
file://...
mailto:
```

For v0.7.31, SBM now allows any URL that Chromium’s URL parser accepts, except for schemes blocked by the new user-controlled protections below.

This avoids manually curating a browser allowlist and should work better across Chromium-based browsers.

### 2. Added user-toggleable protections for risky schemes

Added three new advanced settings, all enabled by default:

```JavaScript
BlockJavascriptBookmarkOpens: true
BlockDataBookmarkOpens: true
BlockBlobBookmarkOpens: true
```

These block extension-initiated opening of:

```
javascript:
data:
blob:
```

Users can disable each one individually under:

```
Options > Advanced > Bookmark URL protections
```

These protections only affect opening bookmarks through SBM. They do not delete, rewrite, or prevent saving existing bookmarks.

### 3. Updated bookmark opening logic

The new logic is centralized in `manager.js`.

Instead of an allowlist of “safe” schemes, SBM now does this:

```JavaScript
function bookmarkOpenProtocolBlocked(protocol) {
  if (protocol === "javascript:") return BlockJavascriptBookmarkOpens;
  if (protocol === "data:") return BlockDataBookmarkOpens;
  if (protocol === "blob:") return BlockBlobBookmarkOpens;
  return false;
}
```

Then bookmark URLs are parsed with `new URL(...)`.

If the URL parses and its scheme is not currently blocked, SBM passes it to Chromium.

### 4. Kept `file:` bookmark support

`file:` URLs are no longer blocked by SBM’s URL-opening filter.

Chromium/browser permissions and user settings may still affect whether the browser itself allows a specific local file URL to open, but SBM no longer blocks it first.

### 5. Added Options UI and translations

Added translated UI strings for the new advanced protection section across:

- `_locales`
- `sbm_locales`

New locale keys include:

```
bookmarkUrlProtections
bookmarkUrlProtectionsHelp
blockJavascriptBookmarkOpens
blockJavascriptBookmarkOpensHelp
blockDataBookmarkOpens
blockDataBookmarkOpensHelp
blockBlobBookmarkOpens
blockBlobBookmarkOpensHelp
urlOpenBlockedByProtection
```

### 6. Warning/error logging

The blocked-open warnings remain routed through `console.warn`, but in `manager.js` this is already captured by SBM’s existing session warning/error log via:

```JavaScript
installConsoleCapture("SBM Manager");
```

So these warnings appear in the existing:

```
Options > Advanced > Debug Settings > Show warnings/errors log
```

I also added error handling around multi-open tab creation in the current window path so failures are reported through the same captured `console.error` route instead of becoming unhandled promise failures.
