Recommended approach: use Chromium’s built-in **`chrome.i18n` localization system** as the foundation, but add an SBM-specific language override setting on top.

Chromium extensions normally localize user-visible strings through a `_locales/<locale>/messages.json` structure, with `"default_locale"` in `manifest.json`, and read strings using `chrome.i18n.getMessage()`. Chrome’s docs describe this as the extension-wide internationalization infrastructure. [Chrome for Developers](https://developer.chrome.com/docs/extensions/reference/api/i18n?utm_source=chatgpt.com)

For SBM, I recommend this structure:

```
simple_bookmarks_manager/
  _locales/
    en/messages.json
    zh_TW/messages.json
    zh_CN/messages.json
    ja/messages.json
    ko/messages.json
    fr/messages.json
    de/messages.json
    es/messages.json
    pt_BR/messages.json
  ...
  i18n.js
  manager.html
  manager.js
  options.html
  options.js
```

Best implementation plan:

1. **Create one central translation key system**

    - Every visible UI string gets a stable key.
    - Example:

          - `appName`
          - `searchPlaceholder`
          - `detailsTitle`
          - `deleteSelected`
          - `openAllNewTab`
          - `optionsUserInterfaceLanguage`
2. **Use Chrome `_locales` for packaged translations**

    - This is the Chromium-standard method.
    - Also lets `manifest.json` localize extension name/description using `__MSG_key__`.
3. **Add SBM’s manual override**

    - Options page gets **User Interface Language**:

          - `Automatic / Browser Default`
          - `English`
          - `繁體中文`
          - `简体中文`
          - `日本語`
          - `한국어`
          - etc.
    - Store this in `chrome.storage.local`.
    - Default = `auto`.
    - If `auto`, use `chrome.i18n.getUILanguage()` / browser UI language where possible.
    - If manually selected, SBM loads that language regardless of browser language.
4. **Do not hard-code text in HTML**

    - Use attributes like:

``` HTML
<span data-i18n="detailsTitle"></span>
<input data-i18n-placeholder="searchPlaceholder">
<button data-i18n="saveButton"></button>
```

1. **Runtime strings in JS also use the same system**

    - Confirmation dialogs, context menus, status text, errors, and generated labels should all use the localization helper.
2. **Fallback chain**

    - Manual selected language.
    - Browser language.
    - Base language match, for example `en_GB → en`.
    - English fallback.
    - Raw key as final fallback for debugging.
